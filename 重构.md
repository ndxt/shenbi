● 目前不支持分屏，是单 canvas 架构。

  现状：
  - TabManager 只有一个 activeTabId，没有 pane group 概念
  - AppShell 只有一个 canvas 渲染槽，activeCanvasRenderer.render(...) 只渲染当前激活的 tab
  - EditorTabs 只有一个横向 tab bar，没有分屏 UI

  要支持分屏，需要改动的层：

  ┌────────────┬─────────────────────────────────────────────────────────────────────────────────────┐
  │     层     │                                        改动                                         │
  ├────────────┼─────────────────────────────────────────────────────────────────────────────────────┤
  │ TabManager │ 引入 pane group 概念，每个 pane 有独立的 activeTabId 和 tab 列表                    │
  ├────────────┼─────────────────────────────────────────────────────────────────────────────────────┤
  │ AppShell   │ canvas 区域从单列改为多 pane 容器，每个 pane 独立渲染一个 canvas，各自有独立的      │
  │            │ canvasDocContext、CanvasSurfaceHandle                                               │
  ├────────────┼─────────────────────────────────────────────────────────────────────────────────────┤
  │ EditorTabs │ 每个 pane 一个 tab bar，支持跨 pane 拖拽                                            │
  ├────────────┼─────────────────────────────────────────────────────────────────────────────────────┤
  │ 持久化     │ 保存/恢复 pane 布局                                                                 │
  └────────────┴─────────────────────────────────────────────────────────────────────────────────────┘

  工作量不小，AppShell 里 canvas
  状态（canvasRuntime、canvasDocContext、canvasSurface）目前都是单例，分屏需要把这些全部 per-pane 化。
